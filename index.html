Salvataggio

Come funziona: chi apre la pagina inserisce solo la propria password: i dati arrivano dal cloud cifrati e vengono sbloccati in locale; la password identifica anche l'utente nei salvataggi e nel log. «Salva su cloud» ricifra e pubblica. Il backup locale (JSON) è in chiaro: conservarlo in un posto sicuro.

Attiva protezione (prima configurazione)

I dati verranno cifrati (AES-256) e sbloccabili con due password personali. La password usata all'accesso determina l'utente. Il token GitHub viene custodito dentro i dati cifrati: nessuno dei due dovrà più incollarlo.
Consiglio: password non banali (almeno 8-10 caratteri): proteggono sia i dati sia il token. Se le dimenticate entrambe, i dati cifrati non sono recuperabili: tenere un backup JSON locale.
